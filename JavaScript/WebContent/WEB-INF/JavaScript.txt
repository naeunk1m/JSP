* TCP/IP프로토콜
 - 애플리케이션층 : 실제 인터넷 응용프로그램
 - 트랜스포트층(TCP/UDP) : 데이터 전송지, 에러검증 및 체크(정정)
 - 네트워크(인터넷)층 (IP) : 패킷단위로 데이터를 정확한 위치에 전달
 - 데이터 링크층 : 랜카드 제어, 데이터 통신을 위한 HW제어

   	 [COM A]				 [COM B]
	애플리케이션층		<->	애플리케이션층
			  (가상적인 데이터 송수신)
	트랜스포트층(TCP/UDP)		트랜스포트층(TCP/UDP)
	네트워크(인터넷)층 (IP)		네트워크(인터넷)층 (IP)
				<->
			 (실질적인 데이터 송수신)	
	데이터 링크층			데이터 링크층


* 서버 :  정보(데이터)를 제공하는 컴퓨터
* 클라이언트 : 정보를 요청/ 제공 받는 컴퓨터
* 호스트(host) : 인터넷상에서의 컴퓨터
* IP 주소 : 인터넷 상에서 컴퓨터를 구분하기위한 정보(전화번호)
              xxx.xxx.xxx.xxx  (0~255)
	ipconfig
	
	localhost : 나 자신(127.0.0.1)
	> ping -t 127.0.0.1

* 포트 번호 : 인터넷 환경에서 사용되는 여러 응용프로그램들을 구분하기위한 번호
  (1024  이전의 번호 사용 X, mysql-3306,oracle-8080,tomcat-8080)
